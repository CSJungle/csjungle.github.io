产品经理如何进行自学

发表于 2020-05-19

如何学习，一直有在考虑，但之前的思考一直太过于详尽了，最后反而觉得乱糟糟的，没有头绪。后来无意间翻到设计师 Melodie 的博文设计师空余时间怎样自学？，突然发现自己想的太过复杂，可以精简一些。
故根据产品经理的特点，自己也简单梳理了一套较为符合自己的流程。

观察

我尝试把观察分为了两种，一种是人，一种是行业(这里其实基本等同于商业)。 个人觉得PM 的工作其实就是基于这两种。
人作为产品的最终触及者，他们的使用方式，认知方式等，都影响着我们该如何去进行产品设计。更不必说像 B 端产品存在使用者和决策者。
行业(or 商业)则是产品本身，一个行业(商业)是如何运行的，它是如何连接上下游的，它是如何获客的，它的优劣势在哪里等等，这些东西对于你在打造自己产品的时候会很有帮助，它山之石可以攻玉。
当然，生活中其他的东西，也可以多观察，时不时会给你一些启发。

阅读

不能停止阅读，包括书籍以及网上一些高质量的文章。
阅读是把别人的知识转化成自己知识的过程。有很多厉害的人，他们把他们的经验，认知总结成文，来进行分享。是很值得一读的。
这个世界上，太阳底下没有新鲜事。大部分出现的事情，其实在以前就已经发生过了。通过阅读，我们可以了解到对方的处理方式，也能知道对方策略下最后产生的结果，错或者对都能让你有所启发。
阅读的分类可能就会比较多了，设计，技术，商业，运营等方面，当然也包括了人文方面的内容，都应该要去阅读。一是这些对产品的建设都有意义，二是这些会作为基石，夯实你的基础，慢慢才会让你可以站的更高。

内化

1.思考

你所看的东西，看了不一定就属于你。要去多去思考。事物背后的原因及逻辑，需要自己去思考，这样才能够成为你自己的东西。

2.输出

要多尝试把自己所学到的东西去输出，输出是一个将自己所学所思整理的过程。
输出自己的所思所学，会有助于将自己的思考进行整理，容易让自己找到自己所缺失的，理解不清晰的，甚至以为自己理解，但其实自己没有办法说清楚的地方。

3.验证

可以尝试去用各种方法验证自己的思考，这样可以尽快的获得自己思考的反馈。有错可以更快的进行修正，没错可以进一步更深入的思考。

4.personal project

通过将自己的一些项目，可以去尝试一些有趣的项目，这种项目会更关注，可能会让自己有更多的思考和脑洞出现。多去享受这种快乐

5.动手实践

其实这个和 personal project很像，但更多的偏向于技术和设计方面。可以尝试去写写 Python 程序，静态页面。虽然有所了解，但总体而言没有去做尝试，确实会存在一些认知差距。

其他

英语和锻炼。
英语有助于保证让自己可以更快更精确的获得一些国外的行业资讯和文章。
锻炼有助于帮助自己可以大部分时间充满精力，也可以较大程度上的让自己身体出问题。
